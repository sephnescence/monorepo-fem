⏺ I found the issue! The problem is in your root tsconfig.json (<PROJECT_ROOT>/monorepo-fem/tsconfig.json:17).

  The config specifies "include": ["src/**/*"] and "rootDir": "./src", but there's no src/ directory at the root level of your monorepo.
  Your actual source code is inside packages/ts-cloudwatch-publisher/src/.

  For a monorepo structure, your root tsconfig.json should typically:
  1. Only define shared compiler options
  2. Not include any files directly (since there's no source code at the root)
  3. Let individual package configs handle their own includes

  The fix: Remove the include, exclude, and rootDir from your root tsconfig.json since it's meant to be a base config only. Here's what
  it should look like:

  {
    "compilerOptions": {
      "target": "ES2022",
      "module": "ES2022",
      "moduleResolution": "bundler",
      "lib": ["ES2022"],
      "strict": true,
      "esModuleInterop": true,
      "skipLibCheck": true,
      "forceConsistentCasingInFileNames": true,
      "resolveJsonModule": true,
      "isolatedModules": true
    }
  }

  Then add the include back to your package-level tsconfig at packages/ts-cloudwatch-publisher/tsconfig.json:

  {
    "extends": "../../tsconfig.json",
    "compilerOptions": {
      "noEmit": false,
      "outDir": "dist",
      "rootDir": "src",
      "declaration": true,
      "declarationMap": true
    },
    "include": ["src/**/*"],
    "exclude": ["node_modules", "dist", ".aws-sam"]
  }
